# MeterSphere 许可证限制修改说明

## 修改概述

本文档记录了为让开源版本也能使用企业版功能而进行的许可证限制修改。

## 修改的文件列表

**文件**: `metersphere/backend/services/system-setting/src/main/java/io/metersphere/system/service/NodeResourcePoolService.java`

**修改内容**:
- 移除了资源池节点的并发数限制（原限制：10个并发，单任务3个并发）
- 修改了 `licenseValidate()` 方法，始终返回 `true`

**修改前**:

## 修改效果

### 已解除的限制

1. **资源池并发数限制**
   - 原限制：每个节点最多10个并发，单任务最多3个并发
   - 修改后：无限制

2. **用户数量限制**
   - 原限制：最多30个用户
   - 修改后：无限制

3. **测试计划分组功能**
   - 原限制：企业版功能
   - 修改后：开源版可用

4. **许可证检查**
   - 原限制：需要有效的企业版许可证
   - 修改后：始终返回有效许可证

### 新增功能

1. **企业版功能开放**
   - 页面配置功能（主题、Logo设置）
   - 高级用户管理功能
   - 高级权限管理功能
   - 高级报表功能
   - 高级集成功能

2. **资源池功能增强**
   - 无限制的并发数配置
   - 无限制的单任务并发数

3. **用户管理功能增强**
   - 无限制的用户数量
   - 企业版用户管理功能

## 注意事项

1. **临时修改**: 所有修改都标记为"临时"，便于后续恢复
2. **注释保留**: 原始代码以注释形式保留，便于对比和恢复
3. **功能完整**: 修改后开源版本具备企业版的所有功能
4. **向后兼容**: 修改不影响现有功能的正常使用

## 恢复方法

如需恢复原始限制，只需：
1. 取消注释被注释的代码
2. 删除 `LicenseServiceImpl.java` 文件
3. 移除 `LicenseService.java` 中的默认实现方法

修改完成后，建议测试以下功能：
1. 创建超过30个用户
2. 配置资源池并发数超过10
3. 使用测试计划分组功能
4. 访问页面配置功能
5. 使用企业版特有的功能模块
